# PRD (Product Requirements Document) - 마이링크

## 1. 프로젝트 개요

**프로젝트명:** 마이링크 (MyLink)
**목표:** 개인 프로필과 주요 링크를 한 곳에 모아 보여주는 링크트리 클론 서비스
**기술 스택:** Next.js, Tailwind CSS, TypeScript

---

## 2. 핵심 기능 (MVP)

| 기능 | 설명 | 우선순위 |
|------|------|----------|
| 프로필 표시 | 이름, 소개, 프로필 이미지 표시 | 높음 |
| 링크 목록 | 클릭 가능한 링크 카드 목록 표시 | 높음 |
| 기술 스택 표시 | 보유 기술 태그 목록 표시 | 중간 |
| 프로젝트 목록 | 진행한 프로젝트 목록 표시 | 중간 |
| 반응형 레이아웃 | 모바일/태블릿/데스크탑 대응 | 높음 |

---

## 3. 사용자 시나리오

### 시나리오 A: 링크를 받은 방문자

1. 사용자가 SNS나 명함에서 마이링크 URL을 받는다
2. 모바일 브라우저로 페이지에 접속한다
3. 프로필 사진과 이름, 소개를 확인한다
4. 원하는 링크 버튼을 클릭해 해당 페이지로 이동한다

### 시나리오 B: 프로필 주인 (본인)

1. 자신의 프로필 페이지를 다른 사람에게 공유하고 싶다
2. 마이링크 URL을 복사해서 SNS 바이오, 명함 등에 붙여넣는다
3. 상대방이 링크를 클릭하면 자신의 프로필과 링크 목록을 확인할 수 있다

## 5. 비기능 요구사항

- 페이지 로딩 속도: 3초 이내
- 반응형 지원: 375px(모바일) ~ 1440px(데스크탑)
- 외부 링크는 새 탭으로 열기 (`target="_blank"`)
- 접근성: 링크에 적절한 `aria-label` 및 `rel="noopener noreferrer"` 적용
